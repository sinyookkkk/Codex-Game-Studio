# Game Design Theory

Use this reference when a task involves game concepts, systems, balance, player psychology, difficulty, UX, or design review.

## MDA

Mechanics are the rules and systems. Dynamics are the behaviors that emerge during play. Aesthetics are the player feelings the design is trying to create.

Use MDA backward:

1. Name the desired aesthetic.
2. Identify the dynamics that create it.
3. Select mechanics that reliably produce those dynamics.
4. Define playtest evidence that can confirm or refute the chain.

## Self-Determination Theory

Check whether the design supports:

- Autonomy: meaningful choices and ownership.
- Competence: readable challenge, feedback, and mastery.
- Relatedness: connection to characters, world, community, or other players.

If a feature does not support at least one of these needs, ask whether it is scope noise.

## Flow

Review flow with these questions:

- Is the goal clear?
- Is feedback immediate enough?
- Does challenge match player skill?
- Can the player recover from mistakes?
- Does the game reduce avoidable friction?

## Bartle Player Types

Use Bartle types as audience hypotheses, not rigid boxes:

- Achievers want goals, mastery, completion, and progress.
- Explorers want hidden systems, discovery, and expressive routes.
- Socializers want communication, shared meaning, and community rituals.
- Competitors want contest, ranking, mastery proof, and mind games.

## Review output

When applying theory, return:

- Target player promise.
- Theory lens used.
- Design risk.
- Evidence needed in playtest.
- Smallest change or test that reduces uncertainty.
