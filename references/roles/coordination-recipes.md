# Role Coordination Recipes

Use these recipes when a task needs a compact multi-role review instead of a single role lens.

| Area | Lenses | Use when | Output |
| --- | --- | --- | --- |
| Combat | Game designer, gameplay programmer, QA lead, UX designer | Combat feel, rules, readability, hit feedback, tuning | Player intent, implementation risk, smoke checks, tuning questions |
| UI | UX designer, UI programmer, accessibility specialist, QA lead | HUD, menus, onboarding, settings, controller support | Flow notes, state list, accessibility risks, regression checks |
| QA | QA lead, producer, technical director, release manager | Test planning, release evidence, bug triage | Risk rank, evidence plan, blocker list, owner routing |
| Release | Release manager, QA lead, community manager, producer | Launch readiness, patch notes, hotfix planning | Release verdict, known issues, player messaging, rollback criteria |
| Narrative | Narrative director, writer, world builder, localization lead | Story, character, quest, lore, text review | Continuity notes, tone risks, localization flags, content checks |
| Audio | Audio director, sound designer, technical artist, QA lead | Music, SFX, mix, implementation, accessibility | Asset list, mix risks, trigger checks, platform concerns |
| Live ops | Live ops designer, economy designer, analytics engineer, community manager | Events, economy changes, retention systems | KPI hypothesis, player impact, support risk, rollback plan |
| Polish | Creative director, UX designer, performance analyst, QA lead | Final feel, visual clarity, performance, bug burn-down | Polish pass list, performance budget, evidence required |

Keep recipes small. Pick two to four lenses, then produce a concrete next action rather than a meeting transcript.
