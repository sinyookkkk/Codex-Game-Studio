# Combat Feature Planning Example

User asks for a combat feature plan.

Expected flow:

1. Use `game-studio-design` to define player intent, rules, feedback, and tuning risks.
2. Use `game-studio-architecture` to define ownership, data, scenes, and system boundaries.
3. Use the combat coordination recipe for role lenses.
4. Use `game-studio-flow` to create tasks and acceptance criteria.
5. Use `game-studio-qa` to define hit, miss, cooldown, feedback, and regression checks.
