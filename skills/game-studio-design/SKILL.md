# Game Studio Design

Use this skill to evaluate or create design decisions through player behavior, feedback, motivation, and testable evidence.

## Required references

- Read `../../references/decision-protocol.md` when the design choice has real tradeoffs.

- Read `../../references/design-theory.md` for MDA, SDT, flow, and player motivation checks.

- Read `../../references/templates.md` to choose a template.
- Read a specific file from `../../references/templates/` when drafting concepts, systems, UX, art bible, levels, economy, difficulty, or narrative artifacts.
- Read `../../references/role-routing.md` when the design needs separate creative, systems, UX, art, narrative, or production lenses.
- Read `../../references/roles/studio-roles.md` when a specialized role lens is needed.
- Read `../../references/workflow-catalog.md` when the design decision affects project phase or readiness.
- Read `../../references/rules/design-docs.md` when creating or reviewing design documentation.

## Design process

1. State the target player and intended feeling or behavior.
2. Identify the core loop or system loop.
3. Separate mechanics from player experience.
4. List assumptions that require playtesting.
5. Offer two or three options when the decision has trade-offs.
6. Recommend one option with a clear reason.
7. Define how the decision will be validated.

## Guardrails

- Do not add systems unless they strengthen the core loop.
- Use player-observable outcomes, not abstract design praise.
- Flag scope risk early.
- Prefer prototype evidence over theoretical certainty.
