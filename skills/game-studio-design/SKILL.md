# Game Studio Design

Use this skill to evaluate or create design decisions through player behavior, feedback, motivation, and testable evidence.

## Required references

- Read `../../references/templates.md` when drafting concepts, systems, or playtest reports.
- Read `../../references/role-routing.md` when the design needs separate creative, systems, UX, or production lenses.
- Read `../../references/studio-workflow.md` when the design decision affects project phase or readiness.

## Design process

1. State the target player and intended feeling or behavior.
2. Identify the core loop or system loop.
3. Separate mechanics from player experience.
4. List assumptions that require playtesting.
5. Offer two or three options when the decision has trade-offs.
6. Recommend one option with a clear reason.
7. Define how the decision will be validated.

## Guardrails

- Do not add systems unless they strengthen the core loop.
- Use player-observable outcomes, not abstract design praise.
- Flag scope risk early.
- Prefer prototype evidence over theoretical certainty.
